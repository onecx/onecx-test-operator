# Architecture: onecx-test-operator

## Overview

`onecx-test-operator` is a Quarkus 3.x backend operator service that executes security and integration tests against running Kubernetes workloads. It orchestrates test execution by inspecting Nginx configurations and probing Quarkus services through their OpenAPI specs.

## Technology Stack

- **Quarkus**: 3.x (parent `onecx-quarkus3-parent:2.4.0`)
- **Java**: 17
- **Build**: Maven ‚â• 3.8, Docker
- **Key libraries**: Fabric8 Kubernetes Client, Vert.x WebClient, MapStruct, Lombok, MicroProfile OpenAPI, Micrometer/Prometheus, OpenTelemetry

## OpenAPI Code Generation

Two server-stub generation executions via `openapi-generator-maven-plugin` (jaxrs-spec / quarkus library):

| Execution ID | Input spec | Generated packages |
|---|---|---|
| `external` | `target/tmp/openapi/onecx-test-operator-v1.yaml` (downloaded from GitHub at `initialize` phase) | `gen.org.tkit.onecx.test.operator.rs.v1` |
| `ui` | `src/main/openapi/onecx-test-operator-ui.yaml` | `gen.org.tkit.onecx.test.operator.ui` |

REST clients (e.g., `QuarkusAdminClient`) are generated by the **quarkiverse codegen** integrated plugin, which consumes OpenAPI specs from `src/main/openapi/`.

## Kubernetes Integration

- Uses `quarkus-kubernetes-client` (Fabric8) to list services/pods and resolve proxy targets.
- `K8sExecService` executes commands inside pods (`nginx -T`) via WebSocket exec ‚Äî **mocked** in tests with `@InjectMock` because `KubernetesMockServer` does not support Vert.x WebSocket.
- Tests use `@WithKubernetesTestServer` for a local Kubernetes mock API.

## Helm Chart

Helm templates located in `src/main/helm/` ‚Äî deployed as a standard Kubernetes operator with a `ServiceAccount`.
